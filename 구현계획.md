# 하이큐 배구게임 구현계획

## 물리 시스템 설계

### 좌표계
- 원점: 캔버스 좌하단 (0, 0)
- Y축: 위 방향 증가 (Canvas 기본값과 반대)
- 단위: 캔버스 가로 = 1
- 캔버스/맵 크기는 사용측에서 지정

예시 (800×600 캔버스):
```
1 unit = 800px
캔버스 높이 = 600/800 = 0.75 unit

Map(1, 1.5) → 캔버스보다 0.75unit 위로 더 큰 물리맵
```

렌더링 변환 (물리 → Canvas 픽셀):
```js
canvas_x =  phys_x * LOGICAL_WIDTH
canvas_y = LOGICAL_HEIGHT - phys_y * LOGICAL_WIDTH
```

---

### 물리맵 (Map)
- `Map(w, h)`로 물리 세계 크기 정의
- 맵 자체가 경계 — 별도 바디 없음
- 경계 restitution 없음 (항상 비탄성)
- 바디는 맵 밖으로 나갈 수 없음

### 엔티티 원점
| 엔티티 | 원점 위치 |
|--------|-----------|
| 공     | 중심점    |
| 플레이어 | 밑바닥 중앙 |
| 네트   | 밑바닥 중앙 |

### 물리 바디 구조
각 엔티티는 바디 목록을 상태(StateSystem)에 보유.  
바디의 위치는 엔티티 원점(전역 좌표) 기준 오프셋.

```js
// 원 (circle)
{ name, shape: 'circle',  ox, oy, r, restitution }

// 캡슐 (capsule) = 선분 + 반지름
{ name, shape: 'capsule', ox, oy, length, angle, r, restitution }
// ox, oy: 캡슐 중심의 원점 오프셋
// angle: 캡슐 방향 (라디안)
// length: 선분 길이
```

### 캐릭터 히트박스 설계

#### 기본 히트박스 (항상 존재)
- 머리: 원
- 몸통: 수직 캡슐

#### 액션별 추가 캡슐 (액션 중에만 추가)
| 액션    | 추가 캡슐            |
|---------|---------------------|
| SPIKE   | 팔 캡슐 (facing 방향, action.tick에 따라 각도 변화) |
| BLOCK   | 팔 캡슐 (위쪽)       |
| DIVE    | 수평 캡슐 (몸 전체)  |
| RECEIVE | 팔 캡슐 (아래 앞쪽)  |

액션별 자세(각도 등)는 `action.tick / action.duration`으로 보간.

```js
// tick() 안에서
if (p.action.type === 'SPIKE') {
  const t = p.action.tick / p.action.duration
  p.bodies.arm.angle = lerp(startAngle, endAngle, t)
} else {
  delete p.bodies.arm
}
```

### 탄성 (Restitution)
바디별 `restitution` 값으로 충돌 처리 방식 결정.

| 조합                        | 처리 |
|-----------------------------|------|
| 탄성(r>0) vs 탄성(r>0)      | 둘 다 반사, 각자의 r 적용 |
| 탄성(r>0) vs 비탄성(r=0)    | 탄성만 반사 (비탄성 속도는 반사 계산 입력으로만 사용), 비탄성은 막힘 |
| 비탄성(r=0) vs 비탄성(r=0)  | 둘 다 법선 방향 속도 → 0 |
| 바디 vs 맵 경계 (탄성)       | 바디 자신의 r로 반사 |
| 바디 vs 맵 경계 (비탄성)     | 법선 방향 속도 → 0 (막힘) |

기본값:
- 공: `restitution = 0.9`
- 플레이어 바디: `restitution = 0`
- 맵 경계: restitution 없음 (항상 비탄성 벽)

---

### 충돌 이벤트
`physics.step(state, dt)`가 충돌 목록을 반환.  
게임 로직 tick()에서 처리 (득점 등).

```js
const events = physics.step(state, dt)
for (const ev of events) {
  // ev: { a: 'ball', b: 'map-floor', normal, point }
  if (ev.a === 'ball' && ev.b === 'map-floor') handleScore(state)
}
```

---

### 이동 범위 제한
- 물리 시스템에서 지원하지 않음
- 게임 로직 tick()에서 처리

```js
// 플레이어가 네트를 넘지 못하도록
p1.x = clamp(p1.x, 0, netX)
p2.x = clamp(p2.x, netX, mapW)
```

## 파일 구조 (예정)

```
src/
├── engine/
│   └── Physics.js        ← 물리 엔진 (순수 함수 + Map 클래스)
│
└── game/
    ├── Hitbox.js         ← 액션별 히트박스 정의
    ├── GameLogic.js      ← tick() 구현 (물리 호출, 이동 제한, 득점 처리)
    └── GameRule.js       ← 매치 설정
```

## Physics.js API (예정)

```js
// 맵 생성
new PhysicsMap(w, h)

// 중력
applyGravity(vy, g)  →  vy + g

// 캡슐 충돌 감지 (공 vs 캐릭터 바디)
detectBallVsCapsule(ball, capsule)  →  { hit, nx, ny, depth } | null
detectBallVsCircle(ball, circle)    →  { hit, nx, ny, depth } | null

// 복합 히트박스 충돌 (가장 깊이 겹친 바디 기준)
detectBallVsCompound(ball, bodies)  →  { hit, nx, ny, depth } | null

// 충돌 해소
resolveCollision(bodyA, bodyB, hitInfo)  →  { velA, velB }

// 맵 경계 처리 (위치 보정 + 속도 처리)
resolveMapBoundary(body, map)  →  { pos, vel, events }
```
